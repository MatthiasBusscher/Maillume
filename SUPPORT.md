# Maillume Support

## Product Questions

Search the [documentation](docs/architecture.md) and existing [GitHub issues](https://github.com/MatthiasBusscher/Maillume/issues) first. After the repository becomes public, use GitHub Discussions for setup questions and ideas that are not yet actionable bugs.

Use the provided issue forms for reproducible bugs, false results, and feature proposals. Include the Maillume version, browser, input method, expected behavior, and safe reproduction steps.

## Protect Your Email Data

Do not attach or paste a real private email, inbox screenshot, raw `.eml` file, private header, sender address, tracking link, API key, password, or log containing scan content. Replace identifying details and reproduce detection problems with synthetic text. See [CONTRIBUTING.md](CONTRIBUTING.md) and [docs/evaluation.md](docs/evaluation.md).

For account or service questions that cannot be discussed publicly, email [support@maillume.io](mailto:support@maillume.io). Do not send suspicious email contents to that address.

## Security And Privacy

- Report vulnerabilities privately through [SECURITY.md](SECURITY.md).
- Send privacy requests to [privacy@maillume.io](mailto:privacy@maillume.io).
- Do not use a public issue for either type of request.

Maillume provides automated risk indicators, not guarantees. For urgent fraud, payment, or account-access concerns, contact the organization through a trusted channel you find independently.
